Use Case 02 · Security Operations

NightShift: Security Investigations That Never Forget

How AION gives a security operations agent a persistent, causally-linked memory of every investigation — so attacker campaigns that span weeks are never lost between shifts.

Primitives9 MCP tools Nodes31 Edges87 OutputSigned incident record + IOC propagation

The Problem: Institutional Amnesia

SOC analysts work in shifts. Alerts open and close. Notes accumulate in tickets that are archived, reassigned, or simply forgotten when a case goes cold. When a new alert fires two weeks later — same IP, different subnet — the next analyst starts from scratch. There is no institutional memory of what that IP did before, who it touched, or what the response was.

Attackers have learned to exploit this. Slow-burn campaigns deliberately spread activity across time boundaries: a failed authentication attempt on Monday, lateral movement on Thursday, data staging over the weekend. Each event looks isolated. The pattern is only visible if you can hold all of it in memory simultaneously — and correlate across the gaps.

NightShift is a security operations agent built on AION. It uses aion_annotate to create a signed, content-addressed node for every observable event. It uses aion_link to build causal chains between those events in real time. It uses aion_recall to surface every relevant prior observation the moment a new signal arrives. The investigation never starts from scratch — it resumes from everything AION already knows.

Initial Detection: Three Events, One Campaign

A failed authentication attempt against a service account triggered NightShift at 02:17 UTC. The agent annotated the event immediately, assigning it a BLAKE3 content address. Within four minutes, a second event arrived: internal lateral movement to a production database using reused credentials. Twenty-two minutes after that, a third: 2.3 GB written to a hidden temporary directory consistent with pre-exfiltration staging.

Each event was annotated as a discrete, signed node before any correlation was attempted. The content address is the node's identity — immutable, verifiable, and chain-linked in the subsequent snapshot.

Causal Chain Construction

With three discrete events annotated, NightShift called aion_link to construct the causal chain. Edge semantics are explicit: each link carries an edge kind (causal or temporal) and a human-readable rationale encoded into the edge metadata. These links are signed alongside the nodes — they cannot be added retroactively without breaking the chain.

The third link — from data-staging back to failed-auth — encodes a cyclical probe pattern. The attacker probed, staged, then probed again from the same source. This cycle is not an error in the graph; it is a finding. aion_contradictions will surface it explicitly in a later pass.

Path Discovery: Tracing the Attacker's Route

aion_path computed the shortest graph path from the attacker's source IP to the suspected exfiltration endpoint. The path traverses exactly 3 hops, crossing two causal edges and one provenance edge. This is the route NightShift uses to construct the incident narrative: a human-readable, graph-verified account of how the attacker moved from initial access to exfiltration.

Policy Action: Blocking the Source IP

With an active exfiltration campaign confirmed, NightShift invoked aion_deny against the attacker's IP node. The denial propagated automatically downstream through the graph: every node reachable from the blocked node inherited the policy restriction. Six downstream nodes were locked. The denial is cryptographically recorded — it appears in the subsequent snapshot digest and cannot be silently reversed.

Recall Across Shifts: The IP Returns

Fourteen days later, a different analyst on a different subnet received a new alert. The source IP was the same: 185.220.101.42. Without AION, this would be treated as a fresh incident — a clean slate. NightShift called aion_recall on the IP string before any other action.

The recall pass propagated spreading activation through the graph from the query term, returning every node with a similarity score above the threshold. The three nodes from the original campaign surfaced immediately — along with the historical note that this IP had been seen 14 days earlier on a different subnet, and that a denial was still active across 6 downstream nodes. The new alert was not a fresh incident. It was a continuation.

Contradiction Detection: The Impossible Claim

The data-staging node carried a metadata annotation from a prior automated scanner: "no external connections detected." But the graph already contained a provenance edge from s3-exfil-endpoint back to data-staging — meaning the exfiltration endpoint was provably written by that process. The two claims are mutually exclusive.

aion_contradictions detected this cycle automatically. The contradiction is not a data quality problem — it is a finding in its own right. The scanner's claim was either wrong, or it was deliberately planted to obscure attribution. Either way, it now appears in the signed incident record, and the signed edge takes precedence over the unsigned scanner annotation.

Snapshot: Sealing the Incident Record

Following all seven analytical passes, NightShift sealed the investigation with aion_snapshot. The snapshot captures the full node and edge state as a BLAKE3-chained, Ed25519-signed digest. This seal is the tamper-evident chain of custody for the incident record — submitted simultaneously to the SIEM, the incident tracker, and the compliance log. Any future analyst querying this campaign finds the sealed snapshot as the authoritative ground truth.

The denial of the source IP is encoded in this digest. The contradiction between the scanner claim and the provenance edge is encoded in this digest. Every causal link is encoded in this digest. Nothing can be silently revised.

Key Findings

  • 01 Three-event causal chain built in real time. Failed authentication, lateral movement, and data staging were causally linked within minutes of each event — before any shift change could break the investigative thread.
  • 02 Blast radius extended to 6 nodes across 4 hops. The database, backup service, audit sink, and exfiltration endpoint were all identified as exposed before any manual triage began. Structural exposure quantified, not estimated.
  • 03 Cross-shift recall surfaced the campaign 14 days later. When the same IP reappeared on a different subnet, NightShift matched it against the prior investigation in a single recall pass — no manual pivoting, no institutional knowledge required.
  • 04 Policy denial propagated to 6 downstream nodes automatically. Blocking the source IP was a single call. The policy layer handled propagation across the graph. The denial is tamper-evident and appears in the snapshot digest.
  • 05 Contradiction between scanner claim and signed edge surfaced automatically. No manual document comparison was needed. The substrate detected the impossible claim against a signed provenance edge and recorded it in the incident record. The signed evidence prevails.
Core Insight
Attackers rely on institutional amnesia. AION removes it. Every IOC, every causal link, every analyst note is signed, content-addressed, and recallable. When 185.220.101.42 appears 14 days later on a different subnet, NightShift already knows who it is, what it did, which nodes it contaminated, and that a denial is still active. The investigation doesn't restart. It resumes.
